# 项目学习指南（以 AI 代码生成工具为例）

## 一、项目概述
- **项目名称**：AI 代码生成助手
- **核心功能**：
    - 根据用户需求生成 HTML/Java/前端代码
    - 结构化输出代码与描述
    - 支持会话记忆，上下文对话
- **技术栈**：Spring Boot + LangChain4j + 通义千问 + MySQL（可选）

## 二、学习目标
1. 掌握 Spring Boot 项目搭建与配置
2. 理解 LangChain4j 实现 AI 结构化输出的原理
3. 学会对接大模型 API（通义千问/DeepSeek 等）
4. 实践接口设计、异常处理、日志规范
5. 了解项目测试与部署流程

## 三、模块拆解
### 1. AI 服务模块
- 定义 AI 服务接口（如 `AiCodeGeneratorService`）
- 用 `record` 定义结构化输出类
- 配置 `ChatLanguageModel` 与 `AiServices` 代理 Bean
- 编写系统提示词（`system-prompt.txt`）

### 2. 接口模块
- 设计 RESTful 接口（`/api/generate/html` 等）
- 接收用户请求参数，调用 AI 服务
- 统一返回封装（`Result<T>`）
- 异常处理与日志记录

### 3. 会话记忆模块（可选）
- 配置 `ChatMemory` 实现上下文记忆
- 支持多用户会话隔离（`@MemoryId`）

### 4. 测试模块
- 单元测试：`@SpringBootTest` 测试 AI 服务
- 接口测试：Postman/Swagger 验证接口
- 性能测试：接口响应时间、并发调用

## 四、学习步骤
1. **环境搭建**：创建 Spring Boot 项目，引入 LangChain4j 依赖
2. **核心配置**：配置大模型 API Key，注册 `ChatLanguageModel` Bean
3. **接口开发**：定义 AI 服务接口与结构化输出类
4. **提示词优化**：编写系统提示词，保证 AI 输出结构化 JSON
5. **测试验证**：编写测试用例，验证代码生成与解析逻辑
6. **扩展功能**：添加会话记忆、代码下载、历史记录等

## 五、常见问题与解决方案
- **AI 输出格式不规范**：在系统提示词中强制要求 JSON 格式，添加示例
- **Token 超限**：限制会话历史长度，精简提示词
- **依赖冲突**：统一版本管理，排查 `pom.xml` 依赖
- **性能瓶颈**：接入 Redis 缓存，异步处理生成任务
