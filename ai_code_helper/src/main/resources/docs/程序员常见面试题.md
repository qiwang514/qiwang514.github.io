# 程序员常见面试题（Java 后端方向）

## Java 基础
1. **== 和 equals() 的区别？**
    - `==` 比较对象内存地址（基本类型比较值）；`equals()` 默认也是比较地址，String/Integer 等类重写后比较内容。
2. **HashMap 的底层原理？**
    - JDK 7：数组+链表；JDK 8+：数组+链表+红黑树；解决哈希冲突、扩容机制、负载因子。
3. **多线程实现方式有哪些？**
    - 继承 Thread、实现 Runnable、实现 Callable + FutureTask、线程池。
4. **volatile 关键字的作用？**
    - 保证可见性、禁止指令重排序，不保证原子性。

## 数据库
1. **MySQL 索引为什么用 B+ 树？**
    - 相比 B 树，B+ 树叶子节点有序、范围查询更快、IO 次数更少。
2. **事务的四大特性（ACID）？**
    - 原子性、一致性、隔离性、持久性。
3. **什么是脏读、不可重复读、幻读？**
    - 脏读：读到未提交数据；不可重复读：同一事务多次查询结果不一致；幻读：范围查询出现新数据。
4. **SQL 优化的常见手段？**
    - 加索引、避免 select *、分页查询、减少子查询、分析慢查询日志。

## Spring 与微服务
1. **Spring IOC 和 AOP 的理解？**
    - IOC：控制反转，容器管理对象生命周期；AOP：面向切面，实现日志、事务等横切逻辑。
2. **Spring Boot 自动配置原理？**
    - `@EnableAutoConfiguration` + `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 加载配置类。
3. **微服务的优缺点？**
    - 优点：独立部署、弹性扩缩、技术栈灵活；缺点：分布式复杂度高、运维成本高。
4. **Redis 分布式锁实现？**
    - SETNX + 过期时间 + Lua 脚本保证原子性，Redisson 封装实现。

## 计算机网络
1. **HTTP 和 HTTPS 的区别？**
    - HTTPS 基于 TLS/SSL 加密，端口 443，HTTP 是明文 80 端口。
2. **TCP 三次握手和四次挥手？**
    - 三次握手建立连接，四次挥手释放连接，保证可靠传输。
3. **Cookie 和 Session 的区别？**
    - Cookie 存在客户端，Session 存在服务端，Session 依赖 Cookie 传递 JSESSIONID。
